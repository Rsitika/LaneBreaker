Q1. Why i am building this project ?
i wanted to built a complete software project using C++. The project demonstrates game development concepts, object-oriented programming, real-time input handling, collision detection, scoring systems, and later API/database integration through a leaderboard system.

Q2. What is laneBreaker ?
LaneBreaker is a lane-based endless bike survival game. The player controls a bike and avoids incoming cars. The game becomes progressively harder as speed increases. The player's objective is to survive as long as possible and achieve a high score.

Q3. What is Raylib?
Raylib is an open-source game development library for C and C++. It provides graphics, input handling, audio support, and other tools required to build games.

Q4. Why did you choose Raylib?
I chose Raylib because it is lightweight, beginner-friendly, easy to learn, and suitable for building 2D games in C++.

Q5. What is a compiler?
A compiler converts human-readable source code into machine code that the computer can execute.

Q6. What is GCC?
GCC is a compiler used to compile C++ programs and generate executable files.

Q7. What is a game loop?
A game loop is the core loop of a game that continuously updates game logic and renders graphics until the game is closed.

Q8. Explain your development workflow.
I write code in VS Code, compile it using GCC through the MSYS2 UCRT64 environment, and then run the generated executable file.

Q9. Why did you use MSYS2?
MSYS2 provided a modern GCC compiler and simplified installation and management of Raylib and other development tools.

Q10. What did you complete on Day 1 of LaneBreaker?
I set up the development environment, installed GCC and Raylib, created the first game window, understood the game loop, and successfully compiled and executed my first Raylib application.
